# 리프트바운드 시뮬레이터 — 서버 실행 가이드 (공개 서버용)

이 폴더(`dist`)를 **통째로** 서버를 돌릴 컴퓨터에 복사하면 됩니다.
Node.js 설치가 **필요 없습니다** (`riftbound-server.exe`에 내장).

> 이 서버는 **데스크톱 클라이언트 프로그램**의 백엔드입니다. 플레이어는 클라이언트
> 앱(RiftboundSim)을 실행한 뒤, 아래에서 안내하는 **접속 주소를 앱에 입력**해 접속합니다.
> (브라우저로 여는 방식이 아닙니다.)

## 1. 서버 켜기

`start-server.bat` 더블클릭 → 검은 창에 접속 주소가 표시되면 실행 중입니다.

- 기본 포트: **8321** (바꾸려면: `start-server.bat 9000` 처럼 인자로 지정)
- 계정/덱은 `data/db.json`에 저장되어 유지됩니다. (비밀번호는 salt+scrypt 해시로만 저장되며 원문은 저장하지 않습니다.)
- 창을 닫으면 서버가 꺼집니다.

## 2. 접속 방법

| 누가 | 주소 |
|---|---|
| 서버 컴퓨터 본인 | `http://localhost:8321` |
| 같은 네트워크(같은 공유기) | `http://<서버 내부IP>:8321` — 실행 창에 표시됨 |
| **다른 네트워크(인터넷)** | 아래 **A(권장)** 또는 B |

### A. 고정 HTTPS 주소 (권장 — 껐다 켜도 주소 동일) ⭐⭐

**불특정 다수 공개 + 항상 같은 주소**를 원하면 이 방법(ngrok 고정 도메인)을 쓰세요.
비밀번호가 암호화(HTTPS)되고, 서버를 껐다 켜도 주소가 바뀌지 않습니다.

최초 1회 설정:
1. https://ngrok.com 무료 회원가입
2. 대시보드 → **Your Authtoken** 복사
3. 대시보드 → **Domains → Create Domain** 으로 무료 고정 도메인 1개 생성
   (예: `my-riftbound.ngrok-free.app`)
4. 이 폴더의 `ngrok-config.txt` 를 열어 채우기:
   ```
   AUTHTOKEN=복사한_토큰
   DOMAIN=my-riftbound.ngrok-free.app
   ```

이후 매번:
1. `start-server.bat` 실행
2. `tunnel-ngrok.bat` 더블클릭 (최초 1회 ngrok 자동 다운로드)
3. **항상 같은** `https://my-riftbound.ngrok-free.app` 주소를 공유

### A-2. 임시 HTTPS 터널 (주소가 매번 바뀜, 설정 불필요)

한두 번 빠르게 열 때만. `start-server.bat` 실행 후 `tunnel.bat` 더블클릭 →
`https://xxxx.trycloudflare.com` (실행할 때마다 주소 변경됨).

### B. 고정 주소 — DuckDNS + 포트포워딩 (제3자 계정 최소, 평문 HTTP) ⚠️

ngrok을 쓰기 싫고 공유기를 직접 설정할 수 있다면 이 방법으로도 고정 주소를
만들 수 있습니다. 집 공인IP가 바뀌어도 DuckDNS 호스트명은 그대로 유지됩니다.
단, 평문 HTTP라 비밀번호 노출 위험이 있어 A 방식보다 안전하지 않습니다.

최초 1회 설정:
1. https://www.duckdns.org 로그인 → 서브도메인 생성 (예: `my-riftbound`)
2. 페이지의 token 복사 → 이 폴더 `ddns-config.txt` 에 입력:
   ```
   DOMAIN=my-riftbound
   TOKEN=duckdns_토큰
   ```
3. 공유기에서 외부 `8321` → 서버 PC 내부IP `8321` (TCP) 포트포워딩
4. 서버 PC 방화벽에서 8321 인바운드 허용 (start-server.bat을 관리자 권한으로
   한 번 실행하면 자동 등록 시도)

이후 매번:
1. `start-server.bat` 실행
2. `ddns-duckdns.bat` 실행 (현재 IP로 5분마다 자동 갱신 — 계속 켜두기)
3. **항상 같은** `http://my-riftbound.duckdns.org:8321` 주소를 공유

### C. 자체 도메인 보유 시 — Cloudflare 네임드 터널 (가장 견고)

소유한 도메인이 있다면 Cloudflare 무료 계정으로 고정 HTTPS 주소를 만들 수 있습니다.
`cloudflared tunnel login` → `cloudflared tunnel create riftbound` →
설정파일(config.yml)에 `hostname: play.내도메인.com`, `service: http://localhost:8321`
지정 → `cloudflared tunnel route dns riftbound play.내도메인.com` →
`cloudflared tunnel run riftbound`. 이후 항상 `https://play.내도메인.com` 로 접속.
(설정이 가장 많지만 가장 안정적)

## 3. 내장된 보안 기능

이 서버는 불특정 다수 공개를 전제로 다음 보호가 적용되어 있습니다.

- **자격증명 보호**: 비밀번호는 salt+scrypt 해시로만 저장. `data/` 폴더와
  서버 소스는 웹으로 절대 노출되지 않음(화이트리스트 서빙).
- **무차별 대입 차단**: IP별 로그인/가입 횟수 제한(rate limit), 계정 유무와
  무관한 일정 응답시간(계정 열거 방지), 비동기 해싱(서버 멈춤 방지).
- **자원 보호**: 계정 수 상한, 요청 크기 제한, WebSocket 메시지 속도 제한,
  토큰 만료(7일), 방 개수 상한.
- **입력 검증**: 덱은 실제 카드 ID·장수 규칙을 서버에서 검증.
- **대전 무결성**: 서버가 발신자 신원/좌석을 확정해, 상대 명의로 행동하거나
  상대 턴을 가로채는 조작을 차단.
- **웹 보안 헤더**: nosniff / X-Frame-Options(클릭재킹 방지) 등.

### 남아있는 한계 (알아두세요)

- 캐주얼 팬 시뮬레이터로 **게임 로직이 각 접속자 클라이언트에서 실행**됩니다.
  따라서 클라이언트를 개조한 상대가 자기 화면에서 카드를 조작하거나
  상대 손패를 계산해 볼 수 있는 가능성은 완전히 막지 못합니다.
  (서버 권위형 게임 엔진이 아니기 때문 — 서로 아는 사람끼리의 캐주얼 대전 권장)
- 관리자 계정/신고 기능은 없습니다. 문제가 되는 계정은 서버를 끄고
  `data/db.json`에서 해당 항목을 지우면 됩니다.

## 4. 파일 구성

| 파일 | 설명 |
|---|---|
| `start-server.bat` | 서버 실행 (exe 우선, 없으면 Node로 실행) |
| `riftbound-server.exe` | 독립 서버 실행 파일 (Node 불필요) |
| `riftbound-server.js` | 서버 번들 (Node 설치 환경용 예비) |
| `tunnel.bat` | HTTPS 인터넷 터널 열기 (Cloudflare) |
| `index.html`, `js/`, `css/` | 게임 클라이언트 |
| `data/db.json` | 계정·덱 데이터 (자동 생성 — 백업하려면 이 파일만 복사) |

## 5. 자주 묻는 질문

- **접속이 안 돼요**: 터널(A) 방식을 먼저 쓰세요. 회사/학교 네트워크에서는
  포트포워딩이 막혀 있을 수 있습니다.
- **카드 이미지가 안 보여요**: 이미지는 라이엇 CDN에서 불러오므로 각 접속자의
  인터넷 연결이 필요합니다.
- **데이터 초기화**: 서버를 끄고 `data/db.json` 삭제.
- **비밀번호를 잊었어요**: 복구 기능이 없습니다. 서버 운영자가 `data/db.json`에서
  해당 계정을 지우면 같은 아이디로 다시 가입할 수 있습니다.

## 모바일 접속 + 접근 권한 관리 (중요)

이 서버는 **게임 웹앱까지 제공**합니다. 따라서 모바일/PC 상관없이:
1. `start-server.bat` 실행 → `tunnel-ngrok.bat`(또는 tunnel.bat)로 `https://…` 주소 생성
2. 그 주소를 **폰 브라우저로 열면** 바로 플레이 (브라우저 메뉴 → "홈 화면에 추가" 하면 앱처럼 사용)
3. PC도 exe 없이 같은 주소로 접속 가능

### 접근 권한을 직접 관리하려면 (접근 코드)
- 이 폴더에 **`access-code.txt`** 파일을 만들고 원하는 코드(예: `우리길드2026`)를 한 줄 적으세요.
- 그러면 **그 코드를 아는 사람만 회원가입(입장)** 할 수 있습니다. 코드는 허용할 사람에게만 알려주세요.
- **권한 회수**: `access-code.txt`의 코드를 바꾸면 이후 신규 가입이 차단됩니다(기존 계정은 유지).
  특정 사람을 완전히 차단하려면 서버를 끄고 `data/db.json`에서 그 계정 항목을 지우세요.
- 파일이 없으면 코드 없이 누구나 가입 가능(공개)합니다. 변경은 30초 내 자동 반영됩니다.
- (환경변수 `ACCESS_CODE`로도 지정 가능)
